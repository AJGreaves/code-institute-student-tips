Troubleshooting Tips

For newbie CI students

Problem

The CSS on your project is not working as you expect it to, or is having no effect at all.

Possible Reason #1

The cache in your browser is not refreshing to your new CSS changes.

How to fix it

When you view your code, always view it in a separate tab. Don't use the preview window as it is buggy:

  • In Cloud9, make sure your html file is open, click “Run” at the top of the page.
  • Click on the link provided at the bottom of the screen and select “Open”.

Now that you are viewing your page in a separate tab:

  • Click Ctrl-f5 on windows or CMD + SHIFT + R on mac.
  • This will force the browser to refresh the cache and display your new CSS changes.
Possible Reason #2

You didn’t save your html file and/or your CSS file before refreshing the browser.

How to fix it
  • View the files in Cloud9, click Ctrl-S on windows or Command+S on Mac to save each file.
  • Then view your page and refresh your browser using Ctrl-f5 on windows or CMD + SHIFT + R on mac.
Possible Reason #3

You forgot to link your CSS file to your html file.

How to fix it
  • In the head of your html file, type link and select “link snippet” from the dropdown that appears as you type.
  • In the href=”” attribute, type the path to your CSS file between the two quotation marks.

(See below for how to find the right file path)

Possible Reason #4

The path to your CSS file in the head of your html file is incorrect.

How to fix it
  • In your cloud9 workspace, look at the folders on the left side of the screen.
  • Right click on your CSS file and select “Copy file path”.
  • Locate the stylesheet link in the head of your html file.
  • Paste the path to your CSS file between the two quotation marks in the href="" attribute of your stylesheet link
  • If Cloud9 adds a / to the beginning of your path, take it out.

Don't forget to save your files before refreshing your browser to check if it worked!
